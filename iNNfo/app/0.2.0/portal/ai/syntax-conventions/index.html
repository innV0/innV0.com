Syntax and Conventions Reference

Sección titulada «Syntax and Conventions Reference»

Purpose: This document provides a complete syntax reference for AI assistants working with iNNfo files and concepts.

Last Updated: 2026-01-22

File Format Overview

Sección titulada «File Format Overview»

iNNfo uses a distributed hierarchical architecture with iNNfo.md files organized in folders.

Architecture

Sección titulada «Architecture»
  • Metamodel: Centralized in root iNNfo.md
  • Relationships: Centralized in root iNNfo.md (graph_edges)
  • Items: Distributed in {Type}/{Item}/iNNfo.md
  • Enrollment: Explicit - only folders with iNNfo.md (or similar) are part of the model

Relationship Definition

Sección titulada «Relationship Definition»

Components:

  1. from: Source type (must exist)
  2. to: Target type (must exist)
  3. label: Relationship identifier (alphanumeric + underscores)
  4. widget: Widget type (see below)
  5. config: Widget-specific configuration

Validation Rules

Sección titulada «Validation Rules»

Root File (iNNfo.md)

Sección titulada «Root File (iNNfo.md)»
  • ✅ category must be model_root
  • ✅ At least one type in metamodel.types
  • ✅ Valid widget types in relationships
  • ✅ graph_edges references existing nodes and labels

Item Files

Sección titulada «Item Files»
  • ✅ category must be item
  • ✅ type must exist in root metamodel
  • ✅ File must be in {TypeName}/{ItemName}/ structure

Relationships

Sección titulada «Relationships»
  • ✅ Source and target nodes must exist
  • ✅ Label must exist in root metamodel
  • ✅ Value must match widget type

For AI Assistants

Sección titulada «For AI Assistants»

Critical Rules

Sección titulada «Critical Rules»
  1. Centralized Metamodel: ALL types and relationships MUST be defined in root iNNfo.md
  2. No Distributed Metamodel: Item files NEVER contain type or relationship definitions
  3. Explicit Enrollment: Only create iNNfo.md for folders that should be in the model
  4. Flat Attachments: Additional files go in same folder as iNNfo.md, not in subfolders
  5. Natural Names: ALWAYS use natural names for types, items, and folders. DO NOT normalize or slugify. E.g., Use “Música Romántica” not “musica-romantica”.

When Creating Models

Sección titulada «When Creating Models»
  1. Start with Root: Always create iNNfo.md first with complete metamodel
  2. Then Items: Create item folders and iNNfo.md files
  3. Add Relationships: Add to graph_edges in root file
  4. Validate: Ensure all references are correct

When Modifying Models

Sección titulada «When Modifying Models»
  1. Metamodel Changes: Edit root iNNfo.md only
  2. Item Changes: Edit item iNNfo.md files
  3. Relationship Changes: Edit graph_edges in root file
  4. Never: Add metamodel to item files
