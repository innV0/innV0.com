Deployment and Releases

Deployment and Releases

Sección titulada «Deployment and Releases»

This guide covers deploying iNNfo to various hosting platforms.

Standalone Distribution Model

Sección titulada «Standalone Distribution Model»

iNNfo is distributed as a single-page application (SPA) that can run entirely in the browser without requiring a server. This approach provides maximum portability and privacy.

How It Works

Sección titulada «How It Works»
  1. Build Process: The application is compiled into a single index.html file (~6.6 MB) using Vite with the vite-plugin-singlefile plugin
  2. All-in-One: JavaScript, CSS, and assets are inlined into the HTML file
  3. Local-First: Works via file:// protocol - no server required
  4. GitHub Pages Deployment: The standalone build is deployed to iNNv0.com via GitHub Pages

Building the Standalone Version

Sección titulada «Building the Standalone Version»

This creates the standalone application in dist-standalone/:

  • index.html - Complete application (~6.6 MB, gzipped: ~1.8 MB)
  • logo.png - Application logo
  • template.nn.md - Default template for new models
  • docs/ - Documentation files
  • skills/ - AI assistant skills

Release Architecture Overview

Sección titulada «Release Architecture Overview»

iNNfo uses a dual-repository architecture:

  • Repo A (Source): Development repository with full source code
  • Repo B (Release): Distribution repository with standalone builds (index.html, viewer.html, templates)

This guide focuses on deploying the development version to hosting platforms. For creating portable, standalone releases, see Release Generation.

Versioning Strategy

Sección titulada «Versioning Strategy»

iNNfo follows Semantic Versioning:

  • MAJOR (1.0.0): Breaking changes
  • MINOR (0.1.0): New features, backward compatible
  • PATCH (0.0.1): Bug fixes

Troubleshooting

Sección titulada «Troubleshooting»

Build fails:

  • Check Node version (requires 20.19+ or 22.12+)
  • Clear node_modules and reinstall
  • Verify all dependencies are compatible

Deployment fails:

  • Check GitHub Actions logs
  • Verify permissions are set correctly
  • Ensure secrets are configured

App doesn’t load:

  • Check base path in vite.config.ts
  • Verify assets are being served correctly
  • Check browser console for errors
